### Training and experiments guide

This document describes how datasets are prepared, how clustering is applied, how to run training (TimeSeq and AASeq), and where outputs are written.

#### Datasets and clustering
- Prepared under `data/dataset/{normal|sampling|customized}/fold_{k}/train_fold.csv, validation_fold.csv, test_fold.csv`.
- Each row corresponds to a peptide-timepoint with lag features and target columns:
  - Lag features: `TurnoverRate_t-1..t-7`, `Label loss_t-1..t-7`, `Label incorporation_t-1..t-7`
  - Targets: `Target_TurnoverRate`, `Target_LabelLoss`, `Target_LabelIncorporation`
  - Metadata: `Cleaned_Peptidoform`, `Protein names`, `UniProt IDs`, `Cluster`, `Peptido_Embedding_Path`
- Cluster labels are assigned by time-series clustering and relabeled by kinetic score (avg_turnover by default) so that `Cluster 0` is the fastest.

#### Visualization and dataset characteristics
- Figures are generated by `scripts/create_dataset.py`:
  - `reports/figures/dataset/timeseries_boxplots_clip_arch.png`
  - `reports/figures/dataset/clustering/cluster_{c}.png`
- Dataset characteristics:
  - `reports/tables/dataset_characteristics.csv`
  - K-fold split summaries: `data/dataset/**/fold_*/stats_fold.csv` and `data/dataset/**/kfold_summary.csv`

#### Running training
- TimeSeq (ESM2 pooled per timepoint):
```bash
python -m src.timeseq \
  --input_type ESM2 --arch Transformer --target both \
  --dataset sampling --folds 10 --epochs 50 --batch_size 8 --lr 1e-3 \
  --clusters 0,2,4   # optional: filter by cluster IDs
```
- TimeSeq (Incorporation-only):
```bash
python -m src.timeseq \
  --input_type ESM2 --arch LSTM --target incorp \
  --dataset sampling --folds 10 --epochs 50 --batch_size 8 --lr 1e-3
```
- AASeq (ESM2 residue embeddings + lag features):
```bash
python -m src.aaseq \
  --input_type ESM2 --arch Transformer --target both \
  --dataset sampling --folds 10 --epochs 50 --batch_size 8 --lr 1e-3 \
  --clusters 0,2,4   # optional
```
- AASeq (Incorporation-only):
```bash
python -m src.aaseq \
  --input_type ESM2 --arch Transformer --target incorp \
  --dataset sampling --folds 10 --epochs 50 --batch_size 8 --lr 1e-3
```

#### Outputs and run IDs
- To avoid overwriting, each run is stored under a timestamped run directory:
  - TimeSeq: `reports/results/timeseq/ESM2_{target}_{arch}_{dataset}/runs/{YYYYMMDD_HHMMSS}/fold_{k}/`
  - AASeq: `reports/results/aaseq/ESM2_{target}_{arch}_{dataset}/runs/{YYYYMMDD_HHMMSS}/fold_{k}/`
- Per-fold files:
  - `run_params.json`: hyperparameters and dataset info (including cluster filters)
  - `test_results.csv`: per-timepoint predictions/targets (+Cluster if available)
  - `cluster_scores.csv`: cluster-wise RMSE/R² (if Cluster column present)
  - AASeq also emits `non_recursive.csv` for backward compatibility; recursive evaluation is saved to `recursive.csv` in some pipelines
- Aggregated across folds (TimeSeq/AASeq): `results.csv`, `avg_results.csv`, `std_results.csv`, `overall_summary.csv`, `summary.md`

#### Cluster filtering
- You can restrict training and evaluation to specific clusters via `--clusters 0,2,4`.
- Internally, splits are filtered before DataLoader creation so that train/val/test all respect the filter.

#### Tips for hard clusters (fast rising / slow turnover)
- Try sampling-based balancing and loss weighting per cluster.
- Add early-interval features: initial slope, early AUC, first/second differences.
- Consider direct turnover head and/or cluster-conditional heads.

#### Reproducibility
- All run parameters are dumped to `run_params.json`. Use this file to reproduce runs.
- Random seeds are not globally fixed; add your seeding strategy if needed.
